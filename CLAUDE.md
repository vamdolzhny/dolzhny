# Security Rules — ОБЯЗАТЕЛЬНО СОБЛЮДАТЬ
*(из презентации курса, слайд 11 — правила для файла CLAUDE.md в корне проекта)*

## Секреты
- НИКОГДА не хардкодить API-ключи, пароли, токены в код
- Всегда использовать .env через python-dotenv / process.env
- Всегда добавлять .env в .gitignore

## База данных
- Только параметризованные запросы (никакого f-string в SQL)
- Никаких деталей об ошибках в HTTP-ответах

## Аутентификация
- Вся логика прав — только на сервере, НИКОГДА в JS
- Хеширование паролей: bcrypt или argon2

## Зависимости
- Использовать только популярные, проверенные пакеты
- Указывать точные версии в requirements.txt / package.json

## Фронтенд
- textContent вместо innerHTML для пользовательских данных
- Никаких eval(), Function() с внешними данными
